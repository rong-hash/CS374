Solution

(a)

Intuition. FifthSort\text{FifthSort}FifthSort divides the array into five segments each of length kkk, placed in five positions.

In the following example, the segments are assumed to initially appear in descending order (or "the furthest away" from their sorted positions).

Initial 5 4 3 2 1
First Sort 3 4 5 2 1
Second Sort 3 4 1 2 5
Third Sort 1 3 4 2 5
Fourth Sort 1 2 3 4 5

Proof. If n<100n < 100n<100, we can always trust the brute-force algorithm.

For n≥100n \ge 100n≥100, we use mathematical induction. Assuming FifthSort\text{FifthSort}FifthSort can sort A[1:k],∀k<mA[1:k], \forall k < mA[1:k],∀k<m, we prove that FifthSort\text{FifthSort}FifthSort can sort A[1:m],m>100A[1:m], m > 100A[1:m],m>100.

  • First Sort on A[1:3k]A[1:3k]A[1:3k] sorts the first 3k3k3k elements

    • Position C holds the largest segment among Segments A, B, C

  • Second Sort on A[2k+1:m]A[2k+1:m]A[2k+1:m] sorts the last m−2km - 2km−2k elements

    • Position E holds the largest segment among Position C, Segments D, E
      • That is, the largest among Segments A, B, C, D, E
    • Position C holds the smallest segment among Segments C, D, E

  • Third Sort on A[1:3k]A[1:3k]A[1:3k] sorts the first 3k3k3k elements

    • Position A holds the smallest segment among Segments A, B, Position C
      • That is, the smallest among Segments A, B, C, D, E

  • Fourth Sort on A[k+1:4k]A[k+1:4k]A[k+1:4k] sorts the middle 3k3k3k elements

    • The whole array is sorted
    • (P.s. Sorting A[k+1:m−k]A[k+1:m-k]A[k+1:m−k] suffices if m is not dividable by 5)

(c) Yes.

If n<13n < 13n<13, we can always trust the brute-force algorithm; same as (b).

For n≥13n \ge 13n≥13, the mathematical induction process in (a) still applies.

Note that kkk not dividable by 5 is not an issue since the indices can handle the edge cases. Adding dummy elements also works
