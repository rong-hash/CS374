Problem 3

The low-degree spanning tree problem. Given a graph GGG and an integer kkk, does GGG contain a spanning tree such that all vertices in the tree have degree at most kkk?

(a) Prove that the low-degree spanning tree problem is NP-hard with a reduction from Hamiltonian path.

(b) Consider the high-degree spanning tree problem. Given a graph GGG and an integer kkk, does GGG contain a spanning tree whose highest degree vertex is at least kkk? Give an efficient algorithm to solve this problem and analyze its time complexity.


Solution

(Adapted from the solution to Problem 3, Lab 20)

(a)

We prove the low-degree spanning tree problem is NP-hard by a reduction from the undirected Hamiltonian path problem.

Given an arbitrary graph GGG, let HHH be the graph obtained by attaching a "fan" of k−2k - 2k−2 edges (with k−2k - 2k−2 new vertices) to every vertex of GGG. I claim that GGG has a Hamiltonian path if and only if HHH has a low-degree spanning tree.

If. Suppose GGG has a Hamiltonian path PPP.

  • Let TTT be the spanning tree of HHH obtained by adding every "fan edge" in HHH to PPP.
  • Every vertex v∈Hv \in Hv∈H is either a leaf of TTT or a vertex of PPP.
  • If v∈Pv \in Pv∈P, then degP(v)≤2\text{deg}_P(v) \le 2degP​(v)≤2, and therefore degT(v)=degP(v)+(k−2)≤k\text{deg}_T(v) = \text{deg}_P(v) + (k-2) \le kdegT​(v)=degP​(v)+(k−2)≤k.

We conclude that HHH has a low-degree spanning tree.

Only if. Suppose HHH has a low-degree spanning tree TTT.

  • The leaves lll of TTT are the vertices of HHH with degree 111, and l∈H−Gl \in H - Gl∈H−G.
  • Let PPP be the subtree of TTT obtained by deleting all the leaves.
  • PPP is a spanning tree of GGG, and for every vertex v∈Pv \in Pv∈P, we have degP(v)=degT(v)−(k−2)≤2\text{deg}_P(v) = \text{deg}_T(v) - (k-2) \le 2degP​(v)=degT​(v)−(k−2)≤2.

We conclude that PPP is a Hamiltonian path in GGG.  ■\ \blacksquare ■

(Page 3 of this source provides another proof without constructing the "fan." kkk is assumed to be greater than 2.)

We claim that GGG has a Hamiltonian path if and only if it has a spanning tree with vertex degree at most 2.

If. If GGG contains a Hamiltonian path, this path must be a spanning tree since the path visits every node, and a path trivially is a tree.

Only if. It is easy to see that such a spanning tree is a Hamiltonian path. Since it has degree at most 2, it cannot branch; since it is spanning, only two vertices can have degree < 2.  ■\ \blacksquare ■

(b)

Pseudocode.

  • Count the degree of each vertex in GGG.
  • If all vertices have degrees lower than kkk, return False\text{False}False.
  • Select any vertex vvv with degree higher than kkk as the root of the high-degree spanning tree. Include all edges incident to vvv in the tree.
  • Perform a depth-first search on GGG starting at vvv and its children.
  • Check if all vertices have been visited (in case there are multiple connected components). If so, return True\text{True}True.

Runtime analysis. Checking the degrees takes O(n)O(n)O(n) time. The DFS takes O(n)O(n)O(n) time. Total runtime is O(n)O(n)O(n).
