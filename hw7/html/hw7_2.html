Problem 2

Given a directed acyclic graph (DAG) G=(V,E)G = (V, E)G=(V,E) with integer (positive or negative) edge weights:

(a) Give an algorithm to find the shortest path from a node sss to another node t.

(b) Give an algorithm to find the longest path from a node sss to another node t.


Solution

(a)

We use a variant of Bellman-Ford. Dynamic programming still helps, but the algorithm can be optimized for a DAG.

DAGSP(V,E,s,t)V′,E′←topoSort(V,E)m←index(s,V′)n←index(t,V′)for i←m+1 to n// initialize the memoization arrayD[i]←∞for i←m to n−1// no need to consider nodes before s or after tfor j←i+1 to ne←(V′[i],V′[j])// for each edge between V′[i] and V′[j]if e∈E′D[j]←min⁡(D[j],D[i]+e.weight)return D[n]\text{DAGSP}(V, E, s, t) \\ \qquad V', E' \gets \text{topoSort}(V, E) \\ \qquad m \gets \text{index}(s, V') \\ \qquad n \gets \text{index}(t, V') \\ \qquad \textbf{for } i \gets m+1 \textbf{ to } n \qquad \text{// initialize the memoization array} \\ \qquad \qquad D[i] \gets \infty \\ \qquad \textbf{for } i \gets m \textbf{ to } n-1 \qquad \text{// no need to consider nodes before s or after t} \\ \qquad \qquad \textbf{for } j \gets i+1 \textbf{ to } n \\ \qquad \qquad \qquad e \gets (V'[i], V'[j]) \qquad \text{// for each edge between } V'[i] \text{ and } V'[j] \\ \qquad \qquad \qquad \textbf{if } e \in E' \\ \qquad \qquad \qquad \qquad D[j] \gets \min(D[j], D[i] + e.\text{weight}) \\ \qquad \textbf{return } D[n] DAGSP(V,E,s,t)V′,E′←topoSort(V,E)m←index(s,V′)n←index(t,V′)for i←m+1 to n// initialize the memoization arrayD[i]←∞for i←m to n−1// no need to consider nodes before s or after tfor j←i+1 to ne←(V′[i],V′[j])// for each edge between V′[i] and V′[j]if e∈E′D[j]←min(D[j],D[i]+e.weight)return D[n]

Runtime analysis. Topological sort takes O(n+m)O(n + m)O(n+m). The nested for\textbf{for}for loop takes O(n2)O(n^2)O(n2) time in the worst case. Total time complexity is O(n2)O(n^2)O(n2).

(b)

The path with the largest length "value" is equivalent to the path with the smallest length value (i.e., the shortest path) on the graph with all edge weights negated. Therefore, a simple adaptation of DAGSP\text{DAGSP}DAGSP suffices. Time complexity is still O(n2)O(n^2)O(n2).

DAGLP(V,E,s,t)for e∈Ee.weight←−e.weightreturn −DAGSP(V,E,s,t)\text{DAGLP}(V, E, s, t) \\ \qquad \textbf{for } e \in E \qquad \\ \qquad \qquad e.\text{weight} \gets -e.\text{weight} \\ \qquad \textbf{return } -\text{DAGSP}(V, E, s, t) DAGLP(V,E,s,t)for e∈Ee.weight←−e.weightreturn −DAGSP(V,E,s,t)
