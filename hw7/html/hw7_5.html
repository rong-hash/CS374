Problem 5

You are given a directed graph G=(V,E)G = (V, E)G=(V,E) with possibly negative weighted edges.

(a) Suppose you know that the shortest path between any two vertices is guaranteed to have at most kkk edges. Give an algorithm that finds the shortest path between two vertices uuu and vvv in O(k(n+m))O(k(n + m))O(k(n+m)) time.

(b) Suppose you know the shortest path from uuu to vvv contains exactly kkk edges. Give an algorithm that finds the shortest path between two vertices uuu and vvv in O(k(n+m))O(k(n + m))O(k(n+m)) time. The path need not be simple.


Solution

(a)

Intuition. Assume the path runs from uuu to vvv. We run the Bellman-Ford algorithm for kkk rounds, and then backtrack to find the distance of vvv from uuu.

To be more specific, we use a modified version of BFS to traverse all the nodes within kkk steps from uuu.

Pseudocode.

  • Create a BFS queue\text{queue}queue, a set visited\text{visited}visited, and a dist\text{dist}dist array initialized with ∞\infty∞ for all vertices except uuu
  • Add uuu to queue\text{queue}queue and mark it as visited\text{visited}visited
  • Initialize an iteration counter
  • While queue\text{queue}queue is not empty AND counter is less than kkk
    • Dequeue a vertex www from queue\text{queue}queue
    • For each vertex xxx adjacent to www (Modification to B-F)
      • dist(x)=min⁡(dist(x),dist(w)+weight(w,x))\text{dist}(x) = \min(\text{dist}(x), \text{dist}(w) + \text{weight}(w, x))dist(x)=min(dist(x),dist(w)+weight(w,x))
      • If dist(x)\text{dist}(x)dist(x) is updated, make www the parent of xxx
        • If xxx is not visited\text{visited}visited, mark xxx as visited\text{visited}visited and add xxx to queue\text{queue}queue.
    • Increment the counter
  • Among all visited\text{visited}visited nodes, find the node vvv and its distance from uuu.
  • Starting at vvv, backtrack the parents until we reach uuu, adding nodes along the way to the path in reverse order.

Time complexity of this modified BFS algorithm is O(k(n+m))O(k(n + m))O(k(n+m)), as it iterates at most kkk times, and each iteration processes all nnn vertices and mmm edges in the worst case.

(b)

Same as part A. We find the distance and path of u→vu \to vu→v after exactly kkk rounds of Bellman-Ford.
