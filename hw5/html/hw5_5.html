Problem 5

A certain string processing language allows the programmer to break a string into two pieces. It costs nnn units of time to break a string of nnn characters into two pieces, since this involves copying the old string.

A programmer wants to break a string into many pieces, and the order in which the breaks are made can affect the total amount of time used. Give a dynamic programming algorithm that takes a list of character positions after which to break and determines the cheapest break cost in O(n3)O(n^3)O(n3) time.


Solution

Preprocessing. Assume the string sss has length nnn and the array of breaking points BBB has length mmm. After breaking, the string is divided into

s[0:B[0]],s[B[0]+1:B[1]],…,s[B[m−1]+1:n−1]s[0 : B[0]], \quad s[B[0] + 1 : B[1]], \quad \ldots, \quad s[B[m-1] + 1 : n-1] s[0:B[0]],s[B[0]+1:B[1]],…,s[B[m−1]+1:n−1]

For convenient indexing, we add −1-1−1 to the beginning and n−1n-1n−1 to the end of the breaking array. Now BBB becomes

{−1,B[0],…,B[m−1],n−1}\{-1, \quad B[0], \quad \ldots, \quad B[m - 1], \quad n-1\} {−1,B[0],…,B[m−1],n−1}

and each subarray has a unified representation of s[B[i]+1:B[i+1]],0≤i≤ms[B[i] + 1 : B[i + 1]], 0 \le i \le ms[B[i]+1:B[i+1]],0≤i≤m.

Recurrence function. Let cost(i,j)cost(i, j)cost(i,j) represents the breaking cost from point iii to point jjj. This function is only defined when i<ji < ji<j. Obviously cost(i,i+1)=0cost(i, i+1) = 0cost(i,i+1)=0 since the substring need not to be cut.

The intuition is that we iterate kkk between (i,j)(i, j)(i,j) and find the breaking point with minimal cost. The algorithm outputs cost(0,m+1)cost(0, m+1)cost(0,m+1) as the final answer.

Time complexity. The time complexity is O(n3)O(n^3)O(n3) since there are n2n^2n2 states (we are filling out a m×mm \times mm×m DP matrix) and each takes O(n)O(n)O(n) time to compute (there's an iteration).
