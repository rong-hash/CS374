Problem 1

Suppose we have a river and on either side are a number of cities numbered from 111 to nnn (North side: N[1…n]N [1 \ldots n]N[1…n], South side: S[1…n]S[1 \ldots n]S[1…n]). The city planner wants to connect certain cities together using bridges and has a list of the desired crossings (xxx is a 2×k2 \times k2×k array where kkk is the number of planned bridges).

Unfortunately, as we know, bridges cannot cross one another over water, so the city planner must focus on building the most bridges from his plan that do not intersect. Describe an algorithm that finds the maximum number of non-intersecting bridges.


Solution

For 1<i<n1 < i < n1<i<n, we sort the tuples (N[i],S[i])(N[i], S[i])(N[i],S[i]) by N[i]N[i]N[i] in ascending order. Then we can find the maximum number of non-intersecting bridges by finding the longest increasing subsequence of S[i]S[i]S[i] with the algorithm from Problem 2.

This algorithm yields valid answers since the LIS in S[i]S[i]S[i] represents the LIS in both N[i]N[i]N[i] (sorted) and S[i]S[i]S[i] at identical indices, which in turn represents the maximum number of bridges whose start and end points are in strictly increasing order.

Sorting takes O(nlog⁡n)O(n \log n)O(nlogn) time, and finding the LIS takes O(n2)O(n^2)O(n2) time (for filling out a n×nn \times nn×n DP matrix). Thus, the total time complexity of our algorithm is O(n2)O(n^2)O(n2).
